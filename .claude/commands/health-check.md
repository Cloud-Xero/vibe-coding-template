# プロジェクト健全性チェック

プロジェクトの全体的な状況を診断してください。

## チェック項目

### 1. ファイル構造
- [ ] 必要な設定ファイルの存在
- [ ] ディレクトリ構造の妥当性
- [ ] 不要ファイルの検出

### 2. 依存関係
- [ ] package.json の整合性
- [ ] 未使用依存関係の検出
- [ ] セキュリティ脆弱性

### 3. Git 状況
- [ ] 追跡されていないファイル
- [ ] 大きなファイルの警告
- [ ] ブランチの状況

### 4. 開発環境
- [ ] 必要なツールのインストール状況
- [ ] 環境変数の設定
- [ ] ポートの競合チェック

### 5. パフォーマンス
- [ ] ディスク使用量
- [ ] ビルド時間の測定
- [ ] メモリ使用量

## 診断結果の報告
- **健全**: 問題なし
- **注意**: 軽微な問題あり
- **警告**: 対応が必要な問題
- **危険**: 即座に対応が必要

各問題に対する具体的な解決策を提示してください。
